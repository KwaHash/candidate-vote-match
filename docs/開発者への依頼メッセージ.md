### ご依頼：政治家向けサイト candidate.seijiselect.jp の本実装（約1ヶ月・MVP）

お世話になっております。
政治家向けサイトのプロトタイプを、**初期リリース向けに機能を絞り込んだMVP**として再設計しました。本実装をお願いしたくご連絡します。
コンセプトは「**政治活動を、ひとつの画面で回す**」。ログイン後トップを「今日やること」にし、メニューは5つに整理しています。
作り直しではなく**完成済みプロトを流用→Supabase接続→Vercel公開**で、1ヶ月程度を目安にしています。

**■ 対象**
- リポジトリ: `candidate-vote-match`（ブランチ `prototype` / タグ `proto-v1`）
- 仕様書: repo内 `docs/実装仕様書_政治家側_1ヶ月版.md`（MVPスコープ・5メニュー・SupabaseSQL・実装順序・受入条件）
- 画面ごとの仕様: `PROTOTYPE_LOG.md` ／ 全体構想: `BACKLOG.md`
- 起動: `npm install` → `npx next dev`（画面は `app/prototype/` 配下、`/prototype` がMVPメニュー）

**■ MVP（5メニューに絞り込み）**
1. 今日やること（ダッシュボード／旧・選挙活動タスクを統合）
2. 政策をつくる（政策スタンス入力／AI政策秘書＝旧AI政策室を統合、1枚政策カード）
3. 発信をつくる（AI政策秘書の発信／LINE・SNS発信）
4. 支援者を管理する（支援者CRM／旧・支援募集を統合）
5. 資金・成果を報告する（政策応援ページ＝旧クラウドファンディング／定期献金管理／支出・証憑（簡易）／成果報告）
- サブ: プロフィール設定・公開LP。

**■ 後回し（初期は非表示・コードは残置）**
候補者AIアバター／AIエージェント・自動化センター／AIツール連携／政策実装スコア／選挙活動タスク（単体）。
→ AIアバターは誤回答・炎上リスクのため初期は出しません。

**■ 進め方（重要）**
- 画面は完成済みなので**作り直さない**でください。
- バックエンドは作らず **Supabase**、公開は **Vercel**。
- 保存処理は **`app/prototype/_store.ts` に集約済み**。**この1ファイルを Supabase に差し替えるだけ**で全画面が繋がります（key→テーブル対応表もファイル内に記載）。
- **AIは「下書き→確認→編集→公開/送信」**。AIが勝手に公開・送信する機能は作らないでください。

**■ 今回やらないこと（別フェーズ）**
- 寄付・献金の**実決済・本人確認・法令対応**（記録・表示のみ）。
- **実AI（LLM）接続**（AI画面はテンプレのまま）。スクレイピング、他サイト統合、後回し機能。

**■ 注意点**
- **お金を実際に動かす機能（寄付・献金決済）は今回入れない**でください（政治資金規正法・本人確認が絡むため別途）。
- 個人情報は Supabase の RLS で「本人のみアクセス」を必ず設定。
- 政治・選挙・献金の各所に**コンプライアンス注意文**を表示（仕様書8章）。価格・実績はサンプル。

**■ 補足**
- 「選挙ごとの候補者一覧のアップロード」は、政治家側ではなく **運営admin** 側で扱う設計です（CSV/手動→審査→承認→公開）。本リポジトリの対象外。

まず仕様書 §「実装順序」の Phase 2 以降（1枚政策カード→支援者CRM強化→政策応援ページ→公開ページ）の見積もり・進め方をご相談させてください。よろしくお願いいたします。
