# 移行ガイド：政治家サイト（プロト `candidate-vote-match` → 本実装 `nominee-vote-match`）

プロトタイプ（本リポジトリ `app/prototype/`）を、本実装リポジトリ `nominee-vote-match`（認証実装済み・Supabase接続済み）へ取り込むための対応表です。

- プロト側: `candidate-vote-match`（`app/prototype/` ＋ `docs/`）
- 本実装側: `nominee-vote-match`（`app/(auth)/` ＋ `utils/supabase/*` 実装済み・ドメイン candidate.seijiselect.jp）
- Supabaseスキーマ: `next-vote-match/supabase/migrations/0001_init.sql`（適用済み）
- 併読: `docs/実装仕様書_政治家側_1ヶ月版.md`、`docs/データ構造_公開質問ボード_共通仕様.md`、`docs/コンプライアンス・リスク一覧.md`

---

## 0. 進め方の原則
1. **画面は作り直さない**：プロトの `app/prototype/<screen>/page.tsx` を本実装へ移し、パスを `/prototype/...` から実運用パスへ。
2. **保存の差し替えが中心**：プロトは `app/prototype/_store.ts` の `loadJSON/saveJSON` に集約。**これを Supabase（`utils/supabase/*`）の select/upsert に置き換える**だけで全画面が繋がる。
3. **認証は実装済み**：`nominee-vote-match` の `utils/supabase/{client,server}` と `middleware.ts` を利用。プロトの `/prototype` 認証バイパス（`providers/auth-provider.tsx` の `AUTH_PATHS`）は**本実装では不要**。
4. **`candidate_id = auth.uid()`** をキーに（RLSが本人限定を担保）。

## 2. 公開質問への回答（他サイト連携の要）
- プロト画面: `app/prototype/public-answers/page.tsx`
- テーブル: **`candidate_question_answers`**（`question_id`・`election_id`・`candidate_id` は国民/運営と同一値。正準スキーマは `docs/データ構造_公開質問ボード_共通仕様.md`）
- 保存キー（プロト）: `_store.ts` の `publicAnswers` → 本番は `candidate_question_answers`（本人のみRLS）
- ステータス（AnswerStatus）: `未回答 / 下書き / 提出済み（運営確認中） / 回答済み（公開） / 回答辞退 / 対象外`。候補者が設定するのは前4つまで。運営adminが「回答済み（公開）」承認。
- 表示先: 国民側「公開質問ボード」の比較表（回答済み/未回答）。運営adminが上位選定・送付・回答管理。

## 3. スキーマに未追加のテーブル（実装時に追加が必要）
`0001_init.sql` に**未含**。expenses/donor-report を実装するなら追加してください：
```sql
create table if not exists finances (           -- 支出・証憑（expenses画面）
  id bigint generated always as identity primary key,
  candidate_id uuid references candidates on delete cascade,
  category text, amount int, date date, vendor text, related_party boolean default false,
  receipt_file text, memo text, created_at timestamptz default now()
);
create table if not exists impact_reports (      -- 寄付者への成果報告（donor-report画面）
  id bigint generated always as identity primary key,
  candidate_id uuid references candidates on delete cascade,
  page_id bigint references policy_support_pages,
  title text, body text, period text, is_published boolean default false, created_at timestamptz default now()
);
-- RLS: いずれも candidate_id = auth.uid() の本人のみ（donations等と同様）
```

## 6. コンプライアンス（本実装でも必ず維持・`docs/コンプライアンス・リスク一覧.md`）
- **お金を動かす機能（寄付・献金の実決済）は入れない**（記録・表示のみ）。
- 献金まわりは注意文＋国籍確認。**外国人・匿名・借名寄付は不可**。候補者本人への直接金銭寄付は原則不可。
- AIは「下書き→本人承認→公開/送信」。**メールでの選挙運動は不可**。
- 個人情報・要配慮個人情報は RLS（本人限定）。RLSは starter のため公開前にレビュー。
