# 実装仕様書（政治家向け candidate.seijiselect.jp / 1ヶ月版）

このドキュメントは、**完成済みプロトタイプ（`prototype` ブランチ）を流用して本実装する**ための仕様書です。
重い機能（実AI・寄付決済・法令対応・スクレイピング）は今回スコープ外。約1ヶ月で公開できる範囲に絞っています。

- 対象リポジトリ: 本リポジトリ `candidate-vote-match`
- ベース: ブランチ `prototype` / タグ `proto-v1`
- 関連: 画面ごとの仕様=`PROTOTYPE_LOG.md`、全体構想・全機能=`BACKLOG.md`

---

## 1. 進め方（重要：作り直さない）
1. **画面は作り直さない。** プロトの Next.js 画面をそのまま使う。
2. **バックエンドは作らない。Supabase**（DB＋認証＋ファイル保存）を使う。
3. **デプロイは Vercel**（GitHub連携で自動公開）。
4. 中心作業は「**`app/prototype/_store.ts` の保存処理を Supabase に差し替える**」＋「**ログイン接続**」。

## 2. 環境・起動
```bash
npm install
npx next dev        # プロト確認（ポートは衝突回避で適宜）
```
- 画面は `app/prototype/` 配下。`/prototype` がプロトタイプ一覧の入口。

## 4. Supabase テーブル（このSQLを実行）
```sql
-- 認証は Supabase Auth。candidates は auth.users と1:1
create table candidates (
  id uuid primary key references auth.users on delete cascade,
  kanji_name text, kana_name text, party text,
  election_type text, district text, status text,
  avatar_url text, education text, career text, political_career text,
  accept_donation boolean default true, support_needs text[],
  updated_at timestamptz default now()
);
create table policy_positions (
  id bigint generated always as identity primary key,
  candidate_id uuid references candidates, issue_id int,
  position text, importance text, evidence_url text, comment text
);
create table campaigns (
  id bigint generated always as identity primary key,
  candidate_id uuid references candidates, title text, theme_id text,
  region_level text, region_name text, goal int, outcome text,
  support_types text[], visibility text, legal_status text,
  created_at timestamptz default now()
);
create table expense_plans (
  id bigint generated always as identity primary key,
  campaign_id bigint references campaigns, purpose text, amount int
);
create table finances (
  id bigint generated always as identity primary key,
  candidate_id uuid references candidates, type text, category text,
  amount int, date date, counterpart text, receipt_url text,
  is_public boolean default true, is_related_party boolean default false
);
create table supporters (
  id bigint generated always as identity primary key,
  candidate_id uuid references candidates, name text, kind text,
  support_types text[], interests text[], region text,
  contact_note text, next_action text, visibility text
);
create table support_requests (
  id bigint generated always as identity primary key,
  candidate_id uuid references candidates, title text, theme_id text,
  need text, detail text, period text, reward text, visibility text
);
create table activities (
  id bigint generated always as identity primary key,
  candidate_id uuid references candidates, label text, points int, date date, memo text
);
create table tasks (
  id bigint generated always as identity primary key,
  candidate_id uuid references candidates, title text, category text, due date, status text
);
-- 定期献金・献金キャンペーン（受け取りはプラットフォーム集約=決済代行を想定。今回は記録のみ）
create table recurring_plans (
  candidate_id uuid references candidates primary key, offered_tiers int[]
);
create table donation_campaigns (
  id bigint generated always as identity primary key,
  candidate_id uuid references candidates, title text, message text,
  goal int, deadline date, channels text[], created_at timestamptz default now()
);
-- AI設定（任意・テンプレ運用のため必須ではない）
create table ai_settings (
  candidate_id uuid references candidates primary key,
  avatar jsonb, ai_tools jsonb, automation jsonb
);
-- 各テーブルに RLS を有効化し「candidate_id = auth.uid() の行のみ読み書き可」を設定
```
- Storage バケット: `avatars`（顔写真）、`receipts`（証憑）。

## 5. 保存処理の差し替え（中心作業）
- プロトの全画面は保存を **`app/prototype/_store.ts` の `loadJSON` / `saveJSON`** 経由で行っている。
- このファイルの中身を Supabase の select / upsert に書き換えるだけで全画面が本番DBへ繋がる（呼び出し側は変更不要）。
- `_store.ts` 内に **保存キー→テーブル対応表** を記載済み。
- プロトは「1キー＝1JSONオブジェクト」保存。Supabaseでは対応テーブルの行に展開して保存する。
- `async` 化する場合は、各画面の `useEffect`（読み込み）に `await` を足す。

## 6. ログイン・ファイル・デプロイ
- **ログイン**: 既存 `app/(auth)/login`・`sign-up` を Supabase Auth に接続。`auth.uid()` を `candidate_id` に使う。本番では `/prototype` の認証バイパス（`providers/auth-provider.tsx` の `AUTH_PATHS`）を戻す。
- **ファイル**: 顔写真→`avatars`、証憑→`receipts` にアップロードしURLを保存（現状はファイル名のみ保持）。
- **デプロイ**: Vercel に接続→環境変数（`NEXT_PUBLIC_SUPABASE_URL` / `NEXT_PUBLIC_SUPABASE_ANON_KEY`）登録→push で自動公開。

## 8. 必須の注意
- **寄付・献金の実決済は今回入れない**（政治資金規正法・本人確認が絡むため別途）。記録・表示のみ。
- 個人情報（支援者情報・思想信条）は **RLS で本人のみアクセス**。JWT等のデフォルト秘密鍵は本番で変更。
- AI出力は「テンプレ（簡易版）」と画面に明記（誤解防止）。
- LP・実績・価格は**サンプル**。中立性（特定政党を推奨しない）を明記。
