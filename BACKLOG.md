# seijiselect.jp プロトタイプ BACKLOG（マスター計画）

「やりたいこと無制限置き場」。ここから数機能を選び `prototype` ブランチで作成 → タグ＋ハンドオフで開発者へ。
**機能はここに無制限に積む。割り込みで今のバージョンに足さない。**（運用ルールは PROTOTYPE_LOG.md 参照）

## 全体コンセプト
3サイトを別々に作らず、**共通の「政策・候補者・寄付・支出・成果データベース」**を持ち、利用者ごとに入口を分ける。

| サイト | 主利用者 | 役割 |
|---|---|---|
| seijiselect.jp | 国民・有権者 | 政策比較、候補者マッチング、寄付、応援、透明化閲覧 |
| candidate.seijiselect.jp | 政治家・候補者・志望者 | 政策登録、クラファン、政治資金透明化、支援者管理 |
| assist.seijiselect.jp | 支援者・専門家・企業 | 寄付、物品貸与、スキル提供、活動支援 |

**共通ID:** User / Candidate / Policy / Campaign / Donation / Allocation / Expense / Impact / Support / Region
→「誰が・何の政策に・いくら出し・誰に届き・何に使われ・どんな成果になったか」を追跡。

## 開発フェーズ（全体ロードマップ）
- **Phase 1 MVP:** 政策一覧・候補者一覧・診断・政策別支援・透明化（各サイトのMVP）
- **Phase 2 透明化強化:** 寄付/支出/成果ID追跡・寄付者別レポート・証憑・月次レポート・地域階層配分
- **Phase 3 政治寄付対応:** 本人確認・国籍確認・寄付上限・直接寄付・収支報告出力・選挙期間制御・法務レビュー
- **Phase 4 AI/マッチング強化:** AI政策秘書・候補者/支援者推薦・配分提案・質問案生成・実装スコア

## トレーサビリティ / 状態管理（Phase2以降）
- ID採番: `DON-2026-000001` / `SUP-…` / `EXP-…` / `IMP-…`
- 寄付状態: received → allocated → reserved → spent → reported → carried_over / refunded / cancelled
- 支出承認フロー: 申請→入力→証憑→一次承認→法務会計チェック→公開範囲→確定→寄付者レポート反映
- 地域階層: national / prefecture / municipality（政策別の初期配分は `app/prototype/_data.ts` に定義）

## 権限設計
guest / voter / supporter / candidate / staff / auditor / admin / legal_reviewer

## 法務・コンプラ（重要・本番で必須）
政治資金規正法・公職選挙法・本人確認・国籍確認・企業団体寄付制限・匿名寄付禁止・寄付上限・選挙期間表示制御・領収書・収支報告CSV・反社/利益相反チェック。
**方針: MVPは政策支援・会費・リソース支援から開始。政治寄付モジュールはPhase3以降（法務確認後）。**

## 自由責任党との接続（段階設計）
Phase1は党を出さず超党派運用 → 段階的に研究会/候補者/明示的支援選択へ。
**原則: 超党派ファンドと自由責任党系支援は明確に分離。本人同意なしに資金・会員情報を共有しない。**
